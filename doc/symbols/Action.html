Action(HTML_Engine_Reference, beforeRemove, serverCall, serverCall)
An action is an object which holds the information regarding the HTML to be displayed, the function to be called when the action is changed. 
		new Action("Actions city", HTML_Engine.selectCity, function() {
			game.worldMap.deselect();
		});
Parameters:
{HTML_Engine} HTML_Engine_Reference
The function which generates the content (usually a HTML_Engine object)
{function} beforeRemove
A function to be called before the Action is done (or removed by the game)
{function} serverCall
If the action needs to get remote information and to execute a function after receiving the inforamation, this value should be an object for the AJAX request. The object MUST contain "type", "url", "data". You can specify a callback function using the parameter "cb"

Method Detail
perform(fade)
If the action needs an AJAX request it does it. Then it calls the call back and updates the HTML. If it does not need an AJAX request, it just updates the HTML
Parameters:
{boolean} fade
If set to true, the action will be shown by a fade effect
remove()
If there was any AJAX request it aborts it. If the action has code to be executed before the function is done, it executes it. Then, it disables the HTML listeners by calling the 'disable' function of the HTML_Engine object
setArguments(args)
It sets the arguments for the action
Parameters:
{object} args
The arguments for the action
update()
It calls the perform method of the action and refreshes the content of the action. It does fadeOUT - fadeIn
